# Vulnbox Configuration Report

## Scope

Single-VPS deployment where Gameserver, WireGuard and team VM containers run on the same host.

## Network Plan

- Gameserver VPN IP: `10.10.0.1`
- Team VM IP pattern: `10.60.<team>.1`
- Player Host IP pattern: `10.81.<team>.<host>`

Configured in: [config/vpn_config.json](/home/akiidjk/WorkSpace/Projects/ctf-gameserver/vulnbox/config/vpn_config.json)

## Generation Artifacts

Generated by `./vulnbox/scripts/setup.sh`:

- WireGuard profiles: `vulnbox/output/wireguard/`
- Multi-team compose: `vulnbox/docker-compose.vms.yml`

Important mode:

- `local_vulnboxes: true`
  - Server `wg0.conf` does not install `10.60.x.1/32` as WG peer routes.
  - VM subnet stays local on Docker bridge.

## Required Runtime Components

- Rootful Docker daemon (`/var/run/docker.sock`) for vulnbox containers.
- WireGuard interface `wg0` on VPS.
- Docker network `vulnbox-vms_vulnbox_vm_net` with subnet `10.60.0.0/16`.

## VPS Routing / Firewall Rules

Applied by: `sudo ./vulnbox/scripts/setup_vps_routing.sh --apply`

The script applies:

1. Kernel networking
- `net.ipv4.ip_forward=1`
- `rp_filter=0` on `all`, `default`, `wg0`, and VM bridge

2. Routing
- Ensures route `10.60.0.0/16 dev <docker-bridge>`

3. raw table
- Deletes conflicting rules matching pattern: drop `10.60.x.1` unless input is docker bridge
- Adds allow rule:
  - `-t raw PREROUTING -i wg0 -d 10.60.0.0/16 -j ACCEPT`

4. filter table
- FORWARD allow:
  - `wg0 -> bridge` (`10.81.0.0/16` to `10.60.0.0/16`)
  - `bridge -> wg0` (`10.60.0.0/16` to `10.81.0.0/16`)
- DOCKER-USER allow for VM subnet in both directions

5. nat table
- SNAT/MASQUERADE host-to-VM path:
  - `-s 10.81.0.0/16 -d 10.60.0.0/16 -o <bridge> -j MASQUERADE`
- VM-to-WG egress masquerade:
  - `-s 10.60.0.0/16 -o wg0 -j MASQUERADE`

## Connectivity Targets

Expected result:

- Player host can ping gameserver and all VMs.
- Gameserver/VPS can ping VMs and hosts.

Validation command:

```bash
./vulnbox/scripts/check_vm_connectivity.sh
```

Deep validation:

```bash
sudo ./vulnbox/scripts/setup_vps_routing.sh --check
sudo wg show
```

## Minimal Bring-up Sequence (from zero)

```bash
./vulnbox/scripts/setup.sh
sudo DOCKER_HOST=unix:///var/run/docker.sock docker compose -f vulnbox/docker-compose.vms.yml up -d --build
sudo cp vulnbox/output/wireguard/gameserver/wg0.conf /etc/wireguard/wg0.conf
sudo wg-quick down wg0 2>/dev/null || true
sudo wg-quick up wg0
sudo ./vulnbox/scripts/setup_vps_routing.sh --apply
```
